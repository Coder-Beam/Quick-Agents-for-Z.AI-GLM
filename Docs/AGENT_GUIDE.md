## 🎯 快速导航

| 使用场景 | 推荐代理 | 调用方式 |
|----------|---------|---------|
| **首次使用/项目初始化** | yinglong-init | `启动QuickAgent` |

## 一、首次使用流程

### 步骤1：启动项目初始化

```
用户：启动QuickAgent
```

**自动触发**：`@yinglong-init`

**执行内容**：
1. 检测项目类型（新项目/现有项目）
2. 检查配置文件（models.json、lsp-config.json）
3. 启动12轮需求澄清
4. 创建项目文档结构
5. 生成任务清单

### 步骤2：配置引导（首次使用）

如果检测到`models.json`或`lsp-config.json`为空，系统会弹出交互式询问卡：

```markdown
## 🔧 首次配置向导

检测到以下配置文件需要补充：
- [ ] models.json（模型配置）
- [ ] lsp-config.json（LSP配置）

### 1. 模型配置

**Q1**: 您主要使用哪个AI模型？
- [ ] GLM-5（推荐，默认）
- [ ] GLM-5-Flash（快速响应）
- [ ] GPT-4o（需自行配置API Key）
- [ ] Gemini 2.0 Flash（需自行配置API Key）
- [ ] 其他（请手动配置）

**Q2**: 是否需要多模型协同？
- [ ] 否，仅使用单一模型（推荐新手）
- [ ] 是，需要智能路由和fallback

### 2. LSP配置

**Q3**: 您的主要开发语言？（可多选）
- [ ] TypeScript/JavaScript
- [ ] Python
- [ ] Rust
- [ ] Go
- [ ] Java
- [ ] C/C++
- [ ] 其他

**Q4**: 是否启用AST-Grep支持？
- [ ] 是（推荐，支持25种语言）
- [ ] 否

---
💡 配置完成后可随时修改：
   - `.opencode/config/models.json`
   - `.opencode/config/lsp-config.json`
```

### 步骤3：开始日常开发

初始化完成后，可以直接与AI对话，系统会自动调度合适的代理。

## 四、Agent 详细说明

### 核心代理（6个）

#### 1. yinglong-init（项目初始化）
- **触发方式**：`启动QuickAgent`
- **适用场景**：首次使用、新项目、项目迁移
- **核心能力**：项目检测、配置引导、需求澄清启动

#### 2. boyi-consult（需求分析）
- **触发方式**：`@boyi-consult`
- **适用场景**：需求澄清、可行性分析、需求文档生成
- **核心能力**：12轮询问、风险评估、专业建议

#### 3. chisongzi-advise（技术推荐）
- **触发方式**：`@chisongzi-advise`
- **适用场景**：技术选型、架构设计、最佳实践
- **核心能力**：技术栈推荐、架构评估、方案对比

#### 4. cangjie-doc（文档管理）
- **触发方式**：`@cangjie-doc`
- **适用场景**：文档创建、文档更新、知识图谱维护
- **核心能力**：文档生成、双向同步、索引维护

#### 5. huodi-skill（技能管理）
- **触发方式**：`@huodi-skill`
- **适用场景**：技能搜索、技能安装、技能优化
- **核心能力**：技能评估、向导式安装、冲突检测

#### 6. fenghou-orchestrate（主调度器）
- **触发方式**：`@fenghou-orchestrate` 或直接对话
- **适用场景**：日常开发、任务分解、代理协调
- **核心能力**：智能调度、并行执行、进度追踪

## 六、常见问题

### Q1: 首次使用应该从哪个Agent开始？
**A**: 直接发送「启动QuickAgent」，系统会自动调用 yinglong-init 开始初始化流程。

### Q2: 日常开发需要手动调用Agent吗？
**A**: 不需要。直接与AI对话即可，系统会智能调度合适的代理。

### Q3: 如何知道当前正在使用哪个Agent？
**A**: 系统会在响应中标注当前调度的代理，如 `[使用 fenghou-orchestrate]`。

### Q4: models.json 和 lsp-config.json 必须配置吗？
**A**: 首次使用时系统会弹出配置引导，按照提示完成即可。之后可随时手动修改。

### Q5: 可以同时调用多个Agent吗？
**A**: 可以使用 `/ultrawork` 命令，系统会自动分解任务并并行调度多个代理。
