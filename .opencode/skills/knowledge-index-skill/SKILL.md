# Knowledge Index Skill

## 描述

此技能用于构建和维护项目知识图谱，提供文档导航、知识关系可视化和快速检索功能，帮助AI和用户快速定位项目信息。

## 使用时机

在以下场景中主动使用此技能：

1. **项目初始化时**：创建项目级INDEX.md知识图谱
2. **创建功能/模块时**：为新功能或模块创建INDEX.md
3. **文档结构变更时**：更新知识图谱的目录结构
4. **新用户加入时**：提供项目文档导航
5. **跨会话衔接时**：快速了解项目结构

## INDEX.md 标准结构

### 1. 文档导航

展示项目目录结构：

```markdown
## 文档导航

### 目录结构

\`\`\`
项目根目录
├── AGENTS.md ──────────── 开发规范
├── 项目需求.md ────────── 原始需求（可选）
└── Docs/
    ├── MEMORY.md ──────── 项目记忆
    ├── TASKS.md ───────── 任务管理
    ├── DESIGN.md ──────── 设计文档
    ├── INDEX.md ───────── 知识图谱
    ├── DECISIONS.md ───── 决策日志
    ├── features/ ──────── 功能文档
    └── modules/ ───────── 模块文档
\`\`\`
```

## 图表类型

### 1. 目录结构图

使用ASCII树形图展示目录结构，清晰展示层级关系。

### 2. 知识关系图

使用Mermaid `graph TD` 展示上下级关系：
- 项目 → 功能 → 模块
- 需求 → 设计 → 实现

### 3. 文档依赖图

使用Mermaid `graph LR` 展示横向依赖：
- 文档间的引用关系
- 模块间的依赖关系

### 4. 实体关系图

使用Mermaid `erDiagram` 展示数据关系：
- 数据实体关系
- 业务对象关系

## 更新规则

### 自动更新时机

1. **新增文档时**：添加到目录结构
2. **文档重命名/移动时**：更新路径引用
3. **功能/模块结构变更时**：更新知识关系图
4. **创建新功能/模块时**：添加到快速参考

### 更新检查项

- [ ] 目录结构是否完整
- [ ] 知识关系图是否准确
- [ ] 快速参考表是否最新
- [ ] 链接是否有效
- [ ] 最后更新时间是否正确

## 维护最佳实践

### 命名规范

- 功能目录：`features/{feature-name}/`
- 模块目录：`modules/{module-name}/`
- 使用kebab-case命名

### 关系维护

1. 保持关系的准确性
2. 及时清理过期关系
3. 避免循环依赖
4. 定期验证链接有效性

### 可视化原则

1. 图表简洁明了
2. 层级不超过4层
3. 使用一致的样式
4. 添加适当的颜色区分

## 注意事项

1. **同步更新**：文档变更时必须同步更新INDEX.md
2. **链接有效**：定期检查所有链接是否有效
3. **层级清晰**：保持知识图谱的层级结构清晰
4. **版本记录**：记录INDEX.md的更新时间
