# Inquiry Card Skill

## 概述

Inquiry Card Skill 是一个用于生成和管理互动询问卡的技能，采用7层扩展模型进行启发式渐进询问。

## 核心能力

### 1. 分层询问模型

基于7层扩展模型进行需求澄清：

| 层级 | 名称 | 核心问题 | 退出标准 |
|------|------|----------|----------|
| L1 | 业务本质 | 为什么做？核心痛点？商业目标？成功指标？ | 目标量化、痛点明确 |
| L2 | 用户画像 | 谁使用？目标用户特征？使用场景？ | 用户定义清晰 |
| L3 | 核心流程 | 完整使用流程？极端场景？异常处理？ | 流程闭环 |
| L4 | 功能清单 | 做什么？功能边界？优先级？ | 功能可量化 |
| L5 | 数据模型 | 数据结构？关系？存储？ | 数据定义完整 |
| L6 | 技术栈 | 前后端框架？数据库？部署？ | 技术选型确定 |
| L7 | 交付标准 | 验收标准？时间节点？质量要求？ | 交付可验证 |

### 2. 询问卡生成

自动生成分层递进式询问卡。

## 使用场景

### 场景1：项目初始化

当用户执行「启动AGENTS.MD」后，自动触发询问卡流程。

### 场景2：需求变更

当用户提出需求变更时，触发针对性询问卡。

### 场景3：功能开发

在开始新功能开发前，确认功能细节。

## 询问卡格式

```markdown
## 第 N 轮询问 - [层级名称]

**当前层级目标**：[本轮询问的核心目标]

**核心问题**：
[使用选项询问卡呈现2-4个相关问题]

**已确认信息**：
- [已确认的关键点1]
- [已确认的关键点2]

**待澄清维度**：
- [x] L1 业务本质
- [ ] L2 用户画像
- [ ] L3 核心流程
- [ ] L4 功能清单
- [ ] L5 数据模型
- [ ] L6 技术栈
- [ ] L7 交付标准
```

## 退出条件

1. **用户确认退出**：用户明确表示「足够」「可以了」「继续」等
2. **7层全部达标**：所有层级的退出标准均已满足
3. **AI建议退出**：AI判断需求已充分明确，询问用户是否确认

## 触发规则

### 自动触发

1. **「启动AGENTS.MD」**：自动启动询问卡流程
2. **需求变更请求**：针对变更内容启动询问

### 手动触发

- 「开始询问」
- 「澄清需求」

## 询问策略

### 递进式原则

1. 从宏观到微观
2. 从业务本质到技术细节
3. 从用户场景到实现方案

### 开放性原则

1. 优先使用开放性问题
2. 引导用户深度思考
3. 避免引导性倾向

### 追根究底原则

1. 对模糊回答继续追问
2. 直到信息完全明确
3. 不接受「大概」「差不多」的回答

## 最佳实践

### 每轮询问

1. 聚焦1-2个核心问题
2. 提供3-5个选项
3. 总结已确认信息
4. 展示进度状态

### 询问节奏

1. 不急于推进
2. 给用户思考时间
3. 根据回答调整策略
4. 灵活回溯补充
