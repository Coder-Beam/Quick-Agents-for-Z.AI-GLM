# Inquiry Card Skill

## 描述

此技能用于生成和管理互动询问卡，采用7层扩展模型进行启发式渐进询问，帮助AI在需求澄清阶段系统性收集信息。

## 使用时机

在以下场景中主动使用此技能：

1. **「启动AGENTS.MD」触发后**：启动完整的7层询问流程
2. **需求变更时**：针对变更内容启动针对性询问
3. **功能开发前**：确认功能细节和边界
4. **技术选型时**：澄清技术约束和偏好
5. **模糊需求出现时**：对任何不明确的信息进行追问

## 7层扩展询问模型

### L1 - 业务本质

**核心问题**：为什么做？核心痛点？商业目标？成功指标？

**退出标准**：目标量化、痛点明确

**典型问题**：
- 这个项目的核心业务目标是什么？
- 当前面临的主要痛点是什么？
- 如何衡量项目的成功？

### L2 - 用户画像

**核心问题**：谁使用？目标用户特征？使用场景？

**退出标准**：用户定义清晰

**典型问题**：
- 目标用户是谁？
- 用户的主要使用场景是什么？
- 用户的技术水平如何？

### L3 - 核心流程

**核心问题**：完整使用流程？极端场景？异常处理？

**退出标准**：流程闭环

**典型问题**：
- 完整的用户操作流程是什么？
- 有哪些极端或边界场景？
- 异常情况如何处理？

### L4 - 功能清单

**核心问题**：做什么？功能边界？优先级？

**退出标准**：功能可量化

**典型问题**：
- 需要实现哪些核心功能？
- 功能的优先级如何排序？
- 哪些功能是必须的，哪些是可选的？

### L5 - 数据模型

**核心问题**：数据结构？关系？存储？

**退出标准**：数据定义完整

**典型问题**：
- 涉及哪些核心数据实体？
- 实体之间的关系是什么？
- 数据量级和存储要求？

### L6 - 技术栈

**核心问题**：前后端框架？数据库？部署？

**退出标准**：技术选型确定

**典型问题**：
- 前端使用什么框架？
- 后端使用什么技术？
- 数据库选型是什么？
- 部署环境有什么要求？

### L7 - 交付标准

**核心问题**：验收标准？时间节点？质量要求？

**退出标准**：交付可验证

**典型问题**：
- 验收标准是什么？
- 有哪些关键时间节点？
- 对性能、安全有什么要求？

## 询问卡格式

每轮询问卡使用以下格式：

```markdown
## 第 N 轮询问 - [层级名称]

**当前层级目标**：[本轮询问的核心目标]

**核心问题**：
[使用question工具呈现2-4个相关问题]

**已确认信息**：
- [已确认的关键点1]
- [已确认的关键点2]
- ...

**待澄清维度**：
- [x] L1 业务本质
- [ ] L2 用户画像
- [ ] L3 核心流程
- [ ] L4 功能清单
- [ ] L5 数据模型
- [ ] L6 技术栈
- [ ] L7 交付标准
```

## 退出条件

满足以下任一条件可退出询问：

1. **用户确认退出**：用户明确表示「足够」「可以了」「继续」等
2. **7层全部达标**：所有层级的退出标准均已满足
3. **AI建议退出**：AI判断需求已充分明确，询问用户是否确认

## 询问原则

### 递进式原则

1. 从宏观到微观
2. 从业务本质到技术细节
3. 从用户场景到实现方案

### 开放性原则

1. 优先使用开放性问题
2. 引导用户深度思考
3. 避免引导性倾向

### 追根究底原则

1. 对模糊回答继续追问
2. 直到信息完全明确
3. 不接受「大概」「差不多」的回答

### 中立性原则

1. 提问保持绝对中立
2. 不带有引导性倾向
3. 让用户自己做决策

## 注意事项

1. **不跳过层级**：按顺序进行，确保每层都达标
2. **灵活回溯**：可根据需要返回之前的层级补充
3. **记录确认**：每轮结束后总结已确认信息
4. **尊重用户**：用户可随时退出或跳过某些问题
