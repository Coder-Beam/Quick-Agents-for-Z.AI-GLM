# SI-Hybrid Skill

阶段-迭代混合模型 (Stage-Iteration Hybrid) - QuickAgents核心开发方法论

---

## 核心定义

### 最小功能单元 (Functional Unit)

| 属性 | 标准 |
|------|------|
| 定义 | 一个功能 = 一个单元 = 一次TDD循环 |
| 开发时间 | ≤ 1.5小时 |
| 代码行数 | ≤ 150行 (不含测试) |
| 测试用例 | ≥ 1个 (覆盖率≥80%) |
| 验收标准 | ≥ 1个可量化指标 |
| 提交粒度 | 1个单元 = 1次提交 |

---

## 简洁命名规范

**原则**: 能看懂 + 能定位 ≠ 完整表达

| 类型 | 长度限制 | 命名风格 |
|------|----------|----------|
| 变量名 | ≤ 15字符 | camelCase |
| 函数名 | ≤ 20字符 | camelCase |
| 类名/接口名 | ≤ 25字符 | PascalCase |
| 文件名 | ≤ 20字符 | kebab-case |
| 常量名 | ≤ 20字符 | UPPER_SNAKE |

**通用缩写**: cfg, ctx, fn, msg, err, val, idx, len, str, num

---

## 二次确认机制 (Double-Check)

**原理**: 同一Prompt接收两次 → 目标更明确

### 触发点

1. **阶段入口确认** - 进入新阶段时重放目标
2. **单元开发前确认** - 开始单元开发时重放验收标准
3. **关键决策确认** - 技术选型/架构变更时
4. **跨会话衔接确认** - 新会话开始时重载上下文

### 单元开发确认模板

```markdown
📋 单元目标确认

单元ID: [U-XXX]
单元名称: [功能名称]
所属模块: [模块名称]
功能描述: [具体功能]
验收标准: [可量化指标]
预计时间: ≤1.5小时

请确认以上信息是否正确? (y/n)
```

## Simplify优化步骤

**位置**: Green之后, Refactor之前

```
TDD循环: Red → Green → Simplify → Refactor
```

### Simplify检查清单

| 类别 | 检查项 | 动作 |
|------|--------|------|
| 过度设计 | 未使用的参数/返回值/抽象层 | 删除/简化/扁平化 |
| 幻觉功能 | 需求外的功能/扩展 | 标记确认 |
| 冗余代码 | 重复逻辑/不必要变量 | 提取/内联/删除 |
| 复杂实现 | 能简单却复杂实现 | 简化 |
| 代码行数 | 函数>50行/文件>200行/嵌套>3层 | 拆分/扁平化 |

---

## 四阶段流程

| 阶段 | 特征 | 退出标准 |
|------|------|----------|
| 阶段1: 需求 | 瀑布(7层询问) | 用户确认REQUIREMENT.md |
| 阶段2: 设计 | 瀑布(架构+模块划分) | 用户确认DESIGN.md |
| 阶段3: 实现 | 敏捷+TDD+Simplify | 所有单元完成+测试通过 |
| 阶段4: 交付 | 敏捷(集成+发布) | 用户验收通过 |

---

## 单元开发循环 (≤1.5小时)

| Step | 名称 | 时间 | 内容 |
|------|------|------|------|
| 🔄 | 二次确认 | 2分钟 | 重放单元目标+验收标准 |
| 1 | 单元设计文档 | 5分钟 | 功能描述+接口定义+验收标准 |
| 2 | Red - 测试先行 | 15分钟 | 编写测试用例，确认失败 |
| 3 | Green - 最小实现 | 25分钟 | 编写最少代码，确认通过 |
| 4 | Simplify - 简化优化 | 15分钟 | 运行检查清单，应用简化 |
| 5 | Refactor - 重构 | 10分钟 | 代码结构优化 |
| 6 | 人类可复现检查 | 10分钟 | 命名+可读性+注释检查 |
| 7 | 原子性提交 | 5分钟 | 提交代码+测试+文档 |
| 8 | 回归测试 | 5分钟 | 运行所有相关测试 |
| 🔄 | 二次确认 | 3分钟 | 单元完成确认 |

---

## 人类可复现检查清单

| 类别 | 标准 |
|------|------|
| 命名规范 | 变量≤15字符, 函数≤20字符, 类≤25字符 |
| 代码可读性 | 函数≤50行, 文件≤200行, 嵌套≤3层 |
| 注释覆盖 | ≥15%, 每个函数有说明, 中文注释 |
| 手动维护 | 人类能理解/修改/复现, 无AI专用语法 |

---

## 回归测试规范

- **触发**: 每个单元/模块完成后
- **范围**: 单元级 → 模块级 → 项目级
- **要求**: 所有测试100%通过, 不允许跳过

---

## 使用方式

### 开始新项目

```
/qa-start
```

AI将执行7层询问模型，收集需求并分解为功能单元。

### 开始单元开发

```
/qa-unit [单元ID]
```

AI将执行二次确认，然后开始TDD循环。

### Simplify检查

```
/qa-simplify
```

AI将对当前代码运行Simplify检查清单。
